# R-1: Tauri release build fails from a clean checkout

## Approach

Updated `Cargo.lock` so the Rust `tauri-plugin-dialog` package resolves to `2.7.0`, matching the existing `@tauri-apps/plugin-dialog` `2.7.0` npm lock. The lock refresh also brings the compatible `tauri-plugin-fs` transitive version selected by the resolver.

Added a Tauri `beforeBuildCommand` that runs the UI production build from the `src-tauri` working directory. This lets `cargo tauri build` produce `ui/dist` itself instead of failing in a clean checkout where the frontend has not already been built.

## Tests

- `cd ui && npm ci`
- `rm -rf ui/dist && cargo tauri build`

## Files changed

- `Cargo.lock`
