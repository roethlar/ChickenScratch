# R-7: Tauri release artifacts lack CI coverage

## Approach

Added a `Tauri Bundles` GitHub Actions workflow that builds the documented desktop release artifacts on their native runner families:

- macOS job builds `cargo tauri build --bundles app,dmg`, verifies the `.app` and `.dmg`, and uploads them as artifacts.
- Linux job installs the Tauri v2 Debian/Ubuntu prerequisites, builds `cargo tauri build --bundles appimage`, verifies an AppImage exists, and uploads it as an artifact.

Both jobs install Node 24, a stable Rust toolchain, frontend dependencies with `npm ci`, and a pinned `tauri-cli` 2.11.1 before building.
