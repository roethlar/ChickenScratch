# L-7: AI SSE streams max-byte guard

## Approach

- Added a shared `ByteBudgetRead` wrapper for streamed AI response bodies.
- Capped raw streamed response bodies at 8 MiB before provider-specific SSE line parsing.
- Applied the budget to Ollama, Anthropic, and OpenAI streaming loops.
- Preserved the existing cancellation checks, request id event routing, and provider-specific stop/error handling.
- Kept non-streaming AI calls unchanged because this finding targets SSE/streaming loops.

## Tests

- `byte_budget_reader_allows_exact_limit_at_eof` verifies an exactly-at-budget response is accepted when EOF follows.
- `byte_budget_reader_rejects_data_after_limit` verifies the first byte beyond the budget returns `InvalidData`.
- `byte_budget_reader_caps_buffered_lines` verifies `BufReader`/line-based SSE parsing cannot read past the budget.

## Known gaps

- This is a conservative fixed budget rather than provider-token-aware sizing. The current 8 MiB cap is far above the expected response size for the configured streaming token limits while still bounding malicious endless streams.
