## R-28: Native Swift and Windows dependency resolution is not locked

### Approach

- Stopped ignoring `macos/Package.resolved` and checked in the SwiftPM lock file so Yams resolves to a pinned version and revision.
- Made the cross-frontend Swift harness run with `--disable-automatic-resolution`, so stale or missing SwiftPM lock data fails instead of silently updating.
- Replaced the floating `Microsoft.WindowsAppSDK` `1.8.*` reference with the resolved `1.8.260508005` package version.
- Enabled NuGet lockfile generation for Windows projects through `windows/Directory.Build.props` and checked in lockfiles for the app, core library, and both Windows harness projects.
- Switched Windows CI and macOS validation restore steps to NuGet `--locked-mode`.
- Added `scripts/check-nuget-package-versions.ps1` to fail CI if Windows package references reintroduce wildcard versions.
- Updated the release runbook to include SwiftPM lock verification and locked NuGet restore.
