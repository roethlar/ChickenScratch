# H-2 — Windows RestoreRevision Forward Commit

## Status

Addressed in branch `fix/h-2-windows-restore-forward-commit`.

## Change

`GitService.RestoreRevision` no longer performs `repo.Reset(ResetMode.Hard, commit)`. It checks out the target commit tree into the working tree, stages the resulting file state, and creates a new restore commit from the current HEAD.

This preserves history: the restore result is a new commit whose parent remains the pre-restore HEAD, while the new commit's tree matches the selected revision.
