# R-26 Release Checksum Gate Path Filter

## Approach

Removed the `paths:` filters from `.github/workflows/validation.yml` for both `push` and `pull_request`. The validation workflow now runs for every push and PR, so the release metadata/checksum check cannot be skipped by docs, license, workflow, or other source-archive-affecting changes.

The Tauri bundle workflow remains scoped to bundle inputs. R-26 is about the checksum validation gate, not artifact smoke-build scheduling.

Reopened fix-up: updated `pkg/arch/PKGBUILD` to the source archive checksum generated for this branch after the workflow change.
