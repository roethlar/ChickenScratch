# H-4: OpenAI Endpoint Validation

## Approach

OpenAI chat completions requests now build a parsed `reqwest::Url` before attaching the bearer token. Empty settings use the default `https://api.openai.com/v1/chat/completions`; custom endpoints must be HTTPS URLs with a host and no username, password, query, or fragment. The helper appends `/v1/chat/completions` for custom base URLs and is used by both streaming and non-streaming OpenAI calls.

Settings save now validates OpenAI endpoints with the same helper, so invalid OpenAI endpoints are rejected before writing settings. Ollama settings are intentionally left alone so local HTTP endpoints such as `http://localhost:11434` and `http://127.0.0.1:11434` remain accepted.

## Known gaps

This patch intentionally does not restrict custom OpenAI-compatible HTTPS hosts, because the existing setting supports configurable OpenAI-style endpoints. It only prevents bearer tokens from being sent to non-HTTPS, credential-bearing, query/fragment-bearing, hostless, or malformed endpoint URLs.

`cargo fmt --package chickenscratch --check` reports existing formatter drift in unrelated `src-tauri` files and lines outside this finding. I did not include those formatter-only changes so the branch stays scoped to H-4.
