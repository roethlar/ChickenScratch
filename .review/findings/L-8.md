# L-8: Hide unsupported Linux AI settings tab

Branch: `fix/l-8-linux-settings-ai-tab`

## Approach

- Inspected the Linux QML settings dialog and Rust bridge settings model.
- Confirmed the Linux bridge round-trips an `ai` JSON section but exposes no Linux AI invokables or UI workflow that consumes those settings.
- Removed the visible AI tab and its stack page from `SettingsDialog.qml`.
- Removed the QML-only `settings.ai` default seeding so the dialog no longer creates editable AI settings for unsupported Linux behavior.

## Tests

- `python3 - <<'PY' ... PY` focused static validation that `SettingsDialog.qml` has no `TabButton { text: "AI" }`, no `settings.ai` bindings, and no AI provider/API key UI strings.
- Attempted `cargo test -p chickenscratch-linux settings_default_round_trip`; it did not reach the test binary on this macOS host because `cxx-qt` fails compiling Homebrew QtCore headers with `qyieldcpu.h:37:5: error: implicitly declaring library function '__yield' with type 'void ()' [-Werror,-Wimplicit-function-declaration]`.

## Files changed

- `linux/qml/SettingsDialog.qml`
- `REVIEW.md`
- `.review/findings/L-8.md`

## Known gaps

- Linux still has `AiSettings` in the Rust settings blob for cross-frontend settings compatibility. It remains non-user-facing in the Linux settings dialog until real Linux AI invokables and feature UI exist.
- Full Linux crate validation remains blocked on this host by the existing Qt/cxx-qt toolchain issue noted above.
