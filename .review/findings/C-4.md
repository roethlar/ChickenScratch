# C-4: Per-project Tauri write locks

## Approach

- Added process-local `ProjectWriteLocks` Tauri state keyed by normalized project path.
- Registered the state in `src-tauri/src/main.rs`.
- Wrapped Tauri commands that mutate project disk state after reading/creating project state, plus git commands that write project-local `.git` state, restore project files, or perform project backup/sync writes.
- Left pure reads and non-project app settings/keyring writes unlocked.

## Mutating Command Audit

Locked:

- `document`: `update_document_content`, `add_comment`, `update_comment`, `delete_comment`, `update_document_metadata`, `rename_node`, `link_documents`, `create_document`, `create_entity`, `create_folder`, `delete_node`, `move_node`
- `project`: `create_project`, `save_project`, `import_scrivener`, `update_project_metadata`
- `threads`: `create_thread`, `update_thread`, `delete_thread`
- `templates`: `create_from_template`
- `io`: `import_file`, `import_markdown_folder`, `record_daily_words`
- `git`: `save_revision`, `restore_revision`, `create_draft`, `switch_draft`, `merge_draft`, `push_backup`, `sync_push`, `sync_fetch`, `restore_document`, `sync_pull`, `sync_abort_pull`, `sync_pull_force`, `backup_on_close`

Intentionally out of scope:

- Pure project reads: `load_project`, `get_document`, `search_project`, `list_threads`, `validate_references`, `save_as_template`, `get_project_stats`, `get_session_progress`, `get_writing_history`
- Git reads/diffs/status: `list_revisions`, `list_drafts`, `sync_status`, `document_history`, `revision_diff`, `word_diff`, `compare_drafts`, `word_diff_drafts`, `has_changes`
- Non-project app state or external output only: app settings/recent-project commands, AI text generation/settings commands, compile output generation, compile format lookup, Pandoc check, Scrivener picker

## Tests

- Added unit tests for same-project serialization and different-project independence in `src-tauri/src/commands/mod.rs`.

## Known gaps

- Command-level concurrent integration tests would need a full Tauri app harness and realistic project fixture. The helper tests cover lock behavior directly.
- Locks are process-local. They do not coordinate separate app instances or external tools.
