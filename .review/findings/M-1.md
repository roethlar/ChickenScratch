# M-1 Git Error Taxonomy

## Approach

- Rebased `fix/m-1-git-error-taxonomy` onto current `master` (`6bdfb09`) before commit.
- Reused the `ChiknError::Git(GitError)` / `GitErrorKind` types already present on current `master`.
- Added a narrow `git2::ErrorCode` / `git2::ErrorClass` / message classifier in `crates/core/src/core/git.rs`.
- Routed only high-value user-actionable paths through the taxonomy: restore revision, current-branch detection, remote push/fetch/pull/force-pull/status/abort, and merge-draft merge/checkout conflict surfaces.
- Kept `ChiknError` serialization as a string, so existing Tauri/CLI callers still receive user-facing error messages.

## Tests

- `cargo test -p chickenscratch-core --test remote_sync`
- Added assertions for `GitErrorKind::NoCommits`, `NoUpstream`, `RemoteUnavailable`, and `NotFastForward`.

## Files changed

- `crates/core/src/core/git.rs`
- `crates/core/src/lib.rs`
- `crates/core/tests/remote_sync.rs`
- `REVIEW.md`
- `.review/findings/M-1.md`

## Known gaps

- This intentionally does not replace every existing `ChiknError::Unknown` in `git.rs`.
- Auth classification is implemented for libgit2 codes/classes/messages but is not integration-tested here because it would require a credentialed or mocked remote transport.
- Tauri still serializes errors as strings; structured UI branching can be added separately without breaking current callers.
