## R-18: Cross-frontend hierarchy tag drift is still hidden by a permissive harness

### Approach

- Updated the Swift reader to decode hierarchy node `type` tags case-insensitively, so Rust canonical `Folder` / `Document` tags and native lowercase tags both load into the same `NodeKind` values.
- Added a Swift regression fixture that opens a Rust-style canonical hierarchy, writes document metadata through ChiknKit, and proves the folder/document ancestry plus document path survive instead of being repaired as root orphans.
- Strengthened the Rust cross-frontend verifier to fingerprint hierarchy document ancestry, id, and path. The harness now records the Rust-converter baseline and requires Swift and C# writer passes to preserve that set exactly.
- Updated the validation workflow so the cross-frontend harness runs with `CHIKN_CROSS_FRONTEND_FAIL_ON_REPAIR=1`, making reader-repair markers a CI failure instead of a local-only stricter mode.
