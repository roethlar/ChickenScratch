# M-6: Tauri close flush

Branch: `fix/m-6-tauri-close-flush`

## Approach

- Replaced the Tauri close path in `App.tsx` with `getCurrentWindow().onCloseRequested`.
- The close handler now awaits `flushPendingEditorSave()` before `backup_on_close`.
- If the editor flush fails, the handler calls `event.preventDefault()` so the Tauri window stays open instead of closing on stale disk state.
- Kept `backup_on_close` failure non-blocking, matching the prior close behavior.
- Kept `beforeunload` as a browser-only best-effort fallback outside Tauri.
