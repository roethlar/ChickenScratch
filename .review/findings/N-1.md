# N-1: `create_entity` bypasses C-3 symlink validation

## Approach

Added `chickenscratch_core::core::project::safe_path` with a shared `ensure_project_subdir_safe` helper that validates component-by-component relative paths, rejects absolute/current/parent components, rejects symlinks and non-directories via `symlink_metadata`, canonicalizes existing directories back under the project root, and creates missing directories one component at a time.

`create_entity` now resolves the requested entity folder through that helper before reading the project. That rejects a hostile `characters` or `locations` symlink before the reader can walk it and before the writer can create entity files. The Tauri command body was extracted to `create_entity_impl` so the filesystem behavior can be tested without constructing `tauri::State`.

Reader required-folder repair now calls the same helper, removing the duplicate safe-folder implementation from `reader.rs`.
