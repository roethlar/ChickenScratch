# L-2: Bundle code splitting

## Approach

- Lazy-loaded non-default app views and panels from `App.tsx`: Corkboard, Preview, Timeline, Revisions, Inspector, command palette, project search, Settings, Stats, Comments, and Compile.
- Changed open-prop modals to conditional rendering so their lazy imports are not fetched on startup.
- Lazy-loaded Binder's per-document `DocumentHistory` modal so revision/history code stays out of the normal Binder path.
- Added Vite manual chunking for major vendor families: React, Tiptap, ProseMirror, Tauri, icons, Zustand, and markdown preview dependencies.

## Tests

- `cd ui && npm run lint && npm run build` - passed.
- `git diff --check` - passed.

## Build output

- Before: one minified `index` JavaScript chunk was roughly 908-911 kB and Vite emitted the large chunk warning.
- After: largest JavaScript chunk is `editor-prosemirror` at 330.04 kB; startup `index` chunk is 49.74 kB; Vite no longer emits the large chunk warning.

## Files changed

- `ui/src/App.tsx`
- `ui/src/components/binder/Binder.tsx`
- `ui/vite.config.ts`

## Known gaps

- The default editor view still loads the editor dependency chunks on startup, because the editor is the first screen for open projects.
- `ui/dist` output is validation output only and is not committed.
