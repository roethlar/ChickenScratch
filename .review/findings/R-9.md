# R-9: Core release validation lacks CI coverage

## Approach

Added a `Validation` GitHub Actions workflow for the non-packaging release gates from `RELEASE.md`:

- Rust format check.
- Rust clippy for the core, Tauri, TUI, and converter packages.
- Rust tests for the same package set.
- UI lint and production build.
- Cross-frontend `.chikn` format harness on a macOS runner with Swift and .NET 10 available.

The cross-frontend step uses a persistent runner temp workdir and asserts `writer-toolchains-ran:2/2` in the manifest so the workflow fails if Swift or C# silently skip.

## Tests

- `ruby -e 'require "yaml"; YAML.load_file(".github/workflows/validation.yml")'`
- `grep -q "cargo fmt --all -- --check" .github/workflows/validation.yml`
- `grep -q "cargo clippy -p chickenscratch-core -p chickenscratch -p chickenscratch-tui -p chikn-converter --all-targets -- -D warnings" .github/workflows/validation.yml`
- `grep -q "writer-toolchains-ran:2/2" .github/workflows/validation.yml`
- `git diff --check`

## Known gaps

The Linux Qt frontend remains excluded from default CI because it requires Qt/cxx-qt native dependencies and has been explicitly out of scope for the root validation suite.
