# R-22 Scrivener Export Path Component

## Approach

`write_scrivx` no longer builds the `.scrivx` filename from `project.name`. It now derives the filename from the selected `.scriv` output path stem and rejects empty, dot, parent-dot, separator-bearing, non-UTF-8, and control-character stems before joining the filename under the export directory.

This keeps a hostile or hand-edited `project.yaml` title such as `../victim/victim` from placing the `.scrivx` file outside the chosen export root.
