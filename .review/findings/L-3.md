# L-3: Modal accessibility

## Approach

- Added a shared `useModalFocusTrap` hook that focuses the modal on open, restores prior focus on close, traps Tab/Shift+Tab inside the modal, closes on Escape, and stops modal key events from reaching app-level shortcuts.
- Wired dialog semantics and focus trapping through the modal overlays: prompt/confirm dialogs, compile, command palette, project search, settings, document history, draft compare, Binder move picker, and merge-conflict resolution.
- Added accessible labels for icon-only close/action buttons and associated compile dialog labels with their inputs.
- Kept the comments panel non-modal because it is a docked side panel, not an overlay. It now uses `role="complementary"` with a labelled header and labelled icon buttons.

## Tests

- `cd ui && npm run lint && npm run build`
- `git diff --check`
- Two subagent review passes on the L-3 branch; first pass found Settings focus timing, modal key propagation, CommandPalette Escape bubbling, and compile labels; second pass found the forward-Tab escaped-focus edge case. All were fixed.
