# R-16 Compile Coverage

## Approach

Added `crates/core/tests/compile_roundtrip.rs` to exercise the compile/export pipeline through the public `compile` entry point with a local pandoc shim. The tests build a synthetic `.chikn` project on disk, compile it through the real project reader/writer path, and assert:

- title-page content, author/title metadata, word-count copy, and `\newpage` are included when requested
- manuscript `compile_order` controls output order
- section separators are inserted between included manuscript documents
- `include_in_compile = false` and non-manuscript documents are excluded
- advertised output formats map to the expected pandoc targets, including `epub` -> `epub3`
- an empty includable manuscript fails cleanly with `ChiknError::InvalidFormat` and does not create an output file

The shim records pandoc arguments and copies the temporary markdown input to the requested output, so the assertions validate ChickenScratch's compile contract without requiring a real pandoc install or format-specific renderers in CI.

## Known gaps

The tests intentionally do not validate PDF rendering fidelity or real pandoc output bytes. That would require a pandoc/LaTeX runtime in CI and belongs in a separate release-packaging smoke test. This branch covers ChickenScratch's compile assembly, filtering, ordering, argument construction, and graceful empty-content failure.
