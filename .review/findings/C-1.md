# C-1: Scrivener Importer UUID Path Traversal

## Approach

Added strict validation for Scrivener `BinderItem` UUID values before they are used as `Files/Data/<uuid>/...` path components. The accepted shape is the Scrivener sample-compatible hyphenated UUID form: 36 ASCII hex/hyphen characters with hyphens at UUID positions. Invalid values fail with `ChiknError::InvalidFormat`.

`get_rtf_path` and `get_media_path` are now fallible and validate before joining paths. `converter/mod.rs` is touched only to propagate those C-1 validation errors through the Scrivener import flow and to test rejection through the real importer path.
