# H-1: Reader Repair Persists Data Loss

## Approach

Changed load-time repair to be in-memory only and non-destructive. Missing document files are now logged and preserved in hierarchy/document references instead of being pruned and persisted back to `project.yaml` during `read_project`.

Required folder repair remains additive, but it now creates folders through a component-safe path helper that rejects symlinks and paths outside the project root. Corrupt `threads.yaml` now fails project load instead of defaulting to an empty thread list that could be saved back over existing thread state.
