## R-17: Remaining git mutation paths bypass dirty-worktree, safe-path, and atomic-write protections

### Approach

- `restore_document` now rejects dirty worktrees before reading historical blobs, then writes restored `.md` and optional `.meta` bytes through a shared writer helper that applies the normal project path validation, symlink checks, and atomic same-directory replacement.
- Added a scoped `pub(crate)` writer helper for raw restored document blobs instead of exposing the writer internals or reserializing historical `.meta` content.
- `create_draft` now rejects dirty worktrees before branch creation, avoiding the orphan-branch case where checkout later fails.
- `switch_draft` rejects dirty worktrees before `set_head`, so unsaved prose cannot be clobbered by a forced checkout.
- `merge_draft` rejects dirty worktrees after no-op analysis but before fast-forward branch movement or three-way merge mutation.
- `sync_pull` rejects dirty worktrees after fetch/up-to-date analysis but before fast-forward branch movement or three-way merge mutation.

### Known Gaps

- `sync_pull` may still fetch remote-tracking refs before rejecting a dirty worktree; this does not overwrite local files or advance the local branch, and keeps no-op/up-to-date pull behavior intact.
- Force pull remains intentionally destructive after confirmation, guarded only against pre-existing dirty worktree state.
