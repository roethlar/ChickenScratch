# N-3: Tightening pass — silent error swallows + perf nits

## Approach

Replaced the two Rust silent filesystem swallows with propagated errors:

- `init_repo` now reports `.gitignore` write failures with the path.
- `push_backup` now reports backup directory creation failures before attempting `Repository::init_bare`.

Bounded `validate_references` entity scanning by extracting `collect_entity_slugs`, limiting each entity type to 1024 documents and at most 8 path components under the entity folder. The same helper preserves existing slug behavior for normal projects.

Documented the reference-validation scope in both `threads.rs` and `docs/UI_CONVENTIONS_NOVELIST.md`: only `pov_character`, `characters_in_scene`, `location`, and `threads` are interpreted by the Tauri validator; other field keys stay opaque.

Timeline parsing now distinguishes "no Story Time values" from "Story Time values exist but none parse", and no longer assigns duration widths when all valid story times have the same timestamp. `SessionBadge` now emits a one-time toast when session progress loading fails, then suppresses repeat toasts on the 30-second refresh loop until the project changes.
