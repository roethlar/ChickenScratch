## R-21: Duplicate document IDs/paths silently alias or overwrite content

### Approach

- Reader hierarchy collection now fails closed on duplicate hierarchy document IDs and duplicate normalized hierarchy document paths instead of silently keeping the first path entry.
- Reader disk traversal now rejects duplicate loaded document IDs and duplicate normalized document paths before inserting into `Project.documents`, preventing later files from overwriting earlier map entries.
- Reader validation now checks hierarchy document nodes against loaded documents: if a hierarchy ID exists on disk, its loaded path must match; if a hierarchy path exists on disk under a different ID, the project is rejected. Missing body files are still allowed so R-19's non-destructive missing-file behavior remains intact.
- Writer validation now rejects document map key/id mismatches and duplicate normalized document paths before rewriting `project.yaml` or any `.md` / `.meta` file.
- Linux native document creation now uses the shared `unique_slug` helper so duplicate titles pick a unique `manuscript/<slug>-N.md` path instead of colliding.
