# L-6: Pandoc path hardening

## Approach

- Removed Tauri's bare `pandoc` / `pandoc.exe` fallback from Pandoc discovery.
- Added a shared `resolve_pandoc` helper that checks an absolute user-configured path first, then absolute standard install locations.
- Made settings validation reject non-empty relative Pandoc paths such as `pandoc`.
- Routed Tauri compile, file import, and Scrivener import through the shared resolver so those user-triggered paths receive an absolute executable path.
- Documented the absolute-path requirement in `docs/USER_GUIDE.md`.
