# H-5: Keyring-backed settings secrets

## Approach

- Added keyring-backed storage for `AiSettings.api_key` and `RemoteSettings.token`, leaving only redacted `null` secret fields plus `api_key_in_keyring` / `token_in_keyring` flags in `settings.json` and public settings APIs.
- AI keys are stored in provider-specific services (`chickenscratch.ai.api_key.{provider}`); the remote token is stored under `chickenscratch.remote.token.sync`.
- Added an internal hydrated settings path for AI and remote git commands that need plaintext credentials at use time.
- Migrates existing plaintext secrets from `settings.json` into the OS keyring and removes them from the file only after successful keyring writes.
- Save semantics preserve existing keyring secrets when the UI sends redacted fields, replace secrets on non-empty input, and delete secrets when configured flags are explicitly cleared or blank secrets are submitted.
- Updated the settings UI to show configured state via placeholders and clear buttons without echoing saved secrets.
- Enabled native keyring backends for macOS Keychain, Windows Credential Store, and Linux keyring/Secret Service.
