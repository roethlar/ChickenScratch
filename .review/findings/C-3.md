# C-3: Symlink Writes Outside Project Root

## Approach

Added component-based document path validation and canonical project-root checks before any project write. `write_project` now validates every document target before rewriting `project.yaml`, so an unsafe document cannot partially rewrite project metadata before failing.

Document writes and deletes now reject symlinked parent directories, symlinked document targets, and symlinked `.meta` targets. Required project folders are also created through the same safe directory path helper instead of blind `create_dir_all` calls.
