# L-1: Binder selector split

## Approach

- Stopped `BinderInner` and `CommandPalette` from subscribing to the full `project` object.
- Split Binder subscriptions into stable project identity fields and hierarchy, using `useShallow` so editor content autosaves do not re-render the binder tree.
- Kept Binder commands reading the current project path from the selected project info instead of from a full-project subscription.
- Moved character/location sections to a derived entity signature so content-only document saves do not re-render those sections when the visible entity list is unchanged.
- Changed thread dots to subscribe to the current document's thread id array and project thread metadata references, avoiding a per-row thread color map rebuild on unrelated autosaves.
- Preserved Flow mode's previous guard against stale hierarchy entries by checking `project.documents` at click time before adding documents to a flow session.

## Known gaps

- Entity sections still scan the document map inside their selector to build a stable signature, so this removes the React re-render work from autosave but does not fully eliminate selector-time scanning. Fully removing that cost would require a normalized project store or a dedicated entity index.
