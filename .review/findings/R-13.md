# R-13: Release metadata check does not compare pinned checksum to archive bytes

## Approach

Tightened `scripts/check-release-metadata.sh` so release mode verifies that the SHA pinned in `pkg/arch/PKGBUILD` equals the source archive produced by `scripts/create-release-source.sh`.

- Default and `--release <version>` checks compare the PKGBUILD pin against an archive generated from `HEAD`.
- `--require-tag` checks the local tag first and, when present, compares against `v<version>` instead of `HEAD`.
- If `--require-tag` is used before the tag exists, the script reports only the missing-tag error and skips the archive comparison.
- Updated the PKGBUILD checksum because the checker script itself is included in the release source archive.
