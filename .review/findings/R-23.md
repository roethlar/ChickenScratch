## R-23: Manual Backup reports success while omitting current work

### Approach

- Added a core `backup_current_work` helper that commits dirty project state before pushing the current branch to the selected backup repository.
- Kept `push_backup` as the low-level push primitive so existing callers and tests keep their narrower behavior.
- Added a Tauri `manual_backup` command that runs under the per-project write lock and calls the commit-then-push helper with a deterministic `Manual backup` revision message.
- Updated the Revisions Backup button to run through `runWithEditorFlush` before invoking `manual_backup`, so pending editor debounce content is flushed before the backend snapshots the project.
- Refresh the revision list after backup so the auto-created manual backup revision is visible immediately.
