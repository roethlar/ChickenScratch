# L-4: Binder keyboard tree

## Approach

- Scoped `role="tree"` to the project hierarchy only, leaving Characters and Locations as separate controls outside the tree.
- Lifted folder open state into `BinderInner` so keyboard navigation can derive a reliable visible-node list that matches the rendered tree.
- Added `role="treeitem"`, `aria-level`, `aria-selected`, `aria-expanded` for expandable folders only, and `role="group"` for open child groups.
- Added tree-container keyboard navigation with `aria-activedescendant`: Up/Down, Home/End, Left/Right open-collapse/parent-child movement, Enter/Space activate, Escape clear, and ContextMenu/Shift+F10 open the node context menu.
- Preserved existing mouse click, drag/drop, context menu, and flow-mode behavior.

## Known gaps

- The `...` row affordance remains mouse-only and hidden from assistive tech; keyboard users use ContextMenu or Shift+F10 on the active tree item instead.
- No browser-level keyboard interaction test harness exists for Binder tree navigation; validation is lint/build plus focused code review.
