# R-8: Release process lacks a canonical runbook

## Approach

Added a root `RELEASE.md` documenting the 1.0 release gate:

- version metadata that must be aligned before tagging;
- required Rust/UI/cross-frontend validation commands;
- macOS, Linux, and Windows artifact build commands;
- tag creation flow;
- Arch package source/checksum update procedure after a real tag exists;
- post-build smoke checks for the highest-risk workflows.

This keeps human-gated release decisions explicit instead of silently changing versions or inventing a checksum before the tag exists.
