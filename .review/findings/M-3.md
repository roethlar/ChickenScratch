# M-3: Pandoc subprocess limits

## Approach

- Added a std-only bounded process helper in `crates/core/src/utils/process.rs`.
- Pandoc calls now run with a 60 second timeout.
- Captured output is capped at 50 MiB combined across stdout and stderr. If the cap is exceeded while reading either stream, the child is killed promptly and the caller receives an explicit error.
- Fixed the four `clippy::io_other_error` warnings from the first review pass by using `io::Error::other`.
- Routed the Rust core compile and Scrivener RTF Pandoc calls through the helper.
- Routed the Tauri import conversion, Pandoc discovery, and settings Pandoc check through the helper.
