# L-5: Swift folder IDs align with Rust UUID roots

## Approach

- Changed Swift project creation to assign UUIDs to the required top-level folders instead of literal ids like `manuscript`, `research`, and `trash`.
- Preserved compatibility for existing callers that still pass legacy root ids by resolving those aliases to the current UUID top-level folder before writing `parent_id` or inserting into the hierarchy.
- Kept disk layout tied to root folder names, so a document created with the legacy `manuscript` alias still lands under `manuscript/`.
- Added Swift check cases for UUID root ids and legacy root alias behavior.
