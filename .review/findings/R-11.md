# R-11: Final 1.0 release metadata is still alpha

## Approach

Prepared the repository metadata for a `1.0.0` tag without creating or pushing the tag.

- Bumped Rust workspace package versions, Tauri app version, Cargo.lock workspace entries, Linux frontend package version, README status, RELEASE.md current version, and Arch `pkgver` to `1.0.0`.
- Replaced README alpha status labels with release/preview wording so the top-level public status no longer advertises the release as alpha.
- Extended the release metadata checker so default mode infers release rules from a non-prerelease version and requires a pinned Arch checksum.
- Added `REVIEW.md export-ignore`, fixed archive mtimes, and archives resolved tree objects so reviewer verdict commits do not affect the deterministic source archive checksum.
- Generated the release source archive from the final tree and pinned its SHA-256 in `pkg/arch/PKGBUILD`.
