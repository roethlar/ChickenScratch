# R-10: Release metadata lacks a deterministic tag and checksum preflight

## Approach

Added executable release metadata tooling so the final 1.0 cut can fail before tagging if version metadata, package source URLs, or Arch checksums drift.

- Added `scripts/check-release-metadata.sh` with a default prerelease check and a stricter `--release <version>` mode.
- Added `scripts/create-release-source.sh` to generate the source tarball that the Arch PKGBUILD downloads from GitHub release assets.
- Updated `pkg/arch/PKGBUILD` to use the real upstream repository, an Arch-safe prerelease `pkgver`, and a release-asset source URL instead of a local placeholder tarball.
- Added `pkg/arch/PKGBUILD export-ignore` so the source archive does not contain the PKGBUILD that checksums it.
- Wired the metadata check into the validation workflow and updated `RELEASE.md` with the checksum-safe source archive sequence.
