# H-3 Dirty Worktree Guards

## Approach

- Added a shared `repo_has_changes` status helper in `crates/core/src/core/git.rs` that includes untracked files and recurses into untracked directories.
- Added `reject_dirty_worktree` and called it before `restore_revision` force-checks out a historical tree.
- Added the same guard at the start of `sync_pull_force`, before fetch/reset, so dirty local content is rejected before the hard reset path.
- Used `ChiknError::Git` with `GitErrorKind::Conflict` because the current taxonomy has no dirty-worktree-specific kind. The message tells the user to save a revision or discard changes first.
