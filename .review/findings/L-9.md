# L-9: Pandoc import output cap

## Approach

- Verified the Tauri file-import Pandoc path already uses the shared bounded process runner introduced for M-3.
- `src-tauri/src/commands/io.rs` now routes `convert_to_markdown_via_pandoc` through `output_bounded(..., PANDOC_OUTPUT_LIMIT_BYTES)`.
- `PANDOC_OUTPUT_LIMIT_BYTES` is a 50 MiB combined stdout/stderr cap, enforced by `crates/core/src/utils/process.rs`; the child process is killed when the cap is exceeded.
- No additional code change is needed for L-9 because the same import callsite named in this finding was fixed by the M-3 implementation.
