# R-14 Native Path Validation

## Approach

Added native safe-path layers for Swift and C# that mirror the Rust writer hardening: document paths must be normal project-relative paths, cannot contain `.` or `..`, cannot be rooted, and cannot traverse symlink/reparse-point ancestors or symlinked `.md` / `.meta` targets. The helpers canonicalize under the project root before allowing read/write/delete.

Also hardened the Rust reader path by validating hierarchy document paths and rejecting symlinked project metadata, document roots, nested folders, `.md`, `.meta`, and `threads.yaml` before reading. This closes the read-side bypass that remained after the Rust writer fixes.
