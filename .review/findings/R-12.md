# R-12: macOS DMG release command is not CI-safe

## Approach

Made the macOS DMG release build use Tauri's CI-safe DMG path explicitly.

- Set `CI=true` on the macOS bundle workflow build step.
- Updated `RELEASE.md` to use `CI=true cargo tauri build --bundles app,dmg`.
- Documented why the environment flag is required: it makes Tauri skip Finder AppleScript layout work that fails in headless automation.

## Tests

- `ruby -e 'require "yaml"; YAML.load_file(".github/workflows/tauri-bundles.yml")'`
- `grep -q "CI=true cargo tauri build --bundles app,dmg" RELEASE.md`
- `grep -q "CI: true" .github/workflows/tauri-bundles.yml`
- `cd ui && npm ci`
- `CI=true cargo tauri build --bundles app,dmg`
- `test -d target/release/bundle/macos/ChickenScratch.app`
- `test -n "$(find target/release/bundle/dmg -name 'ChickenScratch_*.dmg' -print -quit)"`
- `git diff --check`
