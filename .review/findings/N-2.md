# N-2: `DocumentHistory` swallows fetch errors silently

## Approach

Split `DocumentHistory`'s revision-loading effect into two observable failure points. The effect already awaits `flushPendingEditorSave()` before loading git history; this branch now reports flush failure as `File history aborted — editor save failed: ...` and returns before querying git. A separate catch around `gitCmd.documentHistory` keeps the revisions list empty but now shows `Failed to load document history: ...` instead of silently presenting an empty history.

Cancellation guards are preserved so stale async results from a closed dialog or changed document do not emit toasts or update state.
