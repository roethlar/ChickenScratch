## R-24: Document/flow switches do not await failed flushes before replacing buffers and clearing dirty state

### Approach

- Added guarded editor navigation helpers that await `flushPendingEditorSave()` before selecting a document, clearing selection, entering flow mode, or exiting flow mode.
- Updated Binder, Corkboard, Project Search, Command Palette, Stats, Timeline, Revisions thread scene links, and the Flow toolbar exit button to use the guarded helpers.
- Changed the Editor document/flow load effect to await its own pending flush before mutating buffer identity refs, replacing content, clearing content, or clearing dirty state.
- Blocked Binder move-to-Trash/permanent-delete operations that affect the active document or an ancestor folder until the active editor buffer flushes successfully.
- Kept search highlight behavior intact by flushing first, then setting the search highlight and selecting the target document only on success.
