# R-5: macOS DMG output is documented but not configured

## Approach

Changed the default Tauri bundle targets from app-only to `["app", "dmg"]` so `cargo tauri build` produces both macOS artifacts promised by the README.

Corrected the README output paths: Tauri writes the `.app` bundle under `target/release/bundle/macos/` and the `.dmg` under `target/release/bundle/dmg/`.

## Tests

- `cd ui && npm ci`
- `cargo tauri build --bundles app,dmg`
- `test -d target/release/bundle/macos/ChickenScratch.app`
- `find target/release/bundle/dmg -name 'ChickenScratch_*.dmg' -print -quit`
- `git diff --check`
