# M-5: Bound simple_word_diff LCS allocation

## Approach

- Added a `SIMPLE_WORD_DIFF_LCS_CELL_CAP` of `1_500 * 1_500` cells before building the dynamic-programming table.
- Replaced the previous per-side 5000-word guard with a checked `(old.len() + 1) * (new.len() + 1)` cell-count guard.
- Large comparisons now return the same coarse deleted/added fallback without allocating the LCS matrix.
- Small comparisons still use the existing LCS path and preserve current output.

## Tests

- `cargo test -p chickenscratch-core simple_word_diff --lib`

- `crates/core/src/core/git.rs`
- `REVIEW.md`
- `.review/findings/M-5.md`

## Known gaps

- Large diffs still return a coarse whole-old/whole-new diff instead of a partial streaming diff. This matches the existing fallback shape while preventing the large allocation.
