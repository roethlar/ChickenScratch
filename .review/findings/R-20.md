## R-20: Linux native delete removes manifest entries only; deleted files resurrect on reload

### Approach

- Added a shared core `project::deletion::delete_node` helper that finds the target subtree, deletes every descendant document `.md` and `.meta` through `writer::delete_document`, then removes the hierarchy node and document-map entries only after file deletion succeeds.
- The helper returns all deleted document IDs so frontends can clear active editor state for documents deleted via a containing folder.
- Updated the Linux bridge `delete_node` invokable to call the shared helper, write the resulting project only after successful file deletion, and clear the active editor when any deleted descendant document was active.
- Removed Linux's local hierarchy-only `remove_from_hierarchy` helper so delete behavior cannot drift back to manifest-only deletion.
