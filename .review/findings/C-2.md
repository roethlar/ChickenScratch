# C-2: Scrivener Importer FileExtension Sanitization

## Approach

Added a Scrivener media extension sanitizer before `<FileExtension>` values are interpolated into either source or destination paths. The sanitizer trims surrounding whitespace, accepts a single leading dot as normalization, and then requires one non-empty ASCII-alphanumeric path component with a bounded length.

The converter now validates media extensions before creating the output project, uses the sanitized value when building Scrivener media source paths, and uses that same sanitized value for imported media document paths. Unsafe values fail with `ChiknError::InvalidFormat` instead of relying on later project-writer path rejection.

## Known gaps

The sanitizer intentionally rejects multi-part extensions such as `tar.gz` because Scrivener media imports only need a single safe extension component for `content.<ext>` paths.
