# M-2: Corrupt Sidecars Fail Loudly

## Approach

- Kept the current fail-loud `threads.yaml` reader behavior and covered it with the existing corrupt-thread regression test.
- Replaced the writer's best-effort `.meta` preservation parse with a strict helper that returns an error on corrupt existing document metadata instead of overwriting it.
- Validated existing `.meta` files before rewriting `project.yaml`, so a corrupt document sidecar does not become a partial project save.
- Replaced `writing-history.json` `unwrap_or_default()` parsing with a strict JSON parser shared by writing-history reads, writes, and session-progress reads.
- Added regression tests for corrupt `.meta` preservation, writing-history JSON parsing, and the session-progress corrupt-history path.

- `cargo test -p chickenscratch writing_history --bins` - passed.
